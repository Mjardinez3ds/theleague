# AGENTS.md — The League (web)

> Mobile-first PWA for our home fantasy football league. Reads ESPN's
> unofficial Fantasy API, ships static JSON, and renders interactive pages
> with Next.js + Tailwind. Companion to the `sportsbook-bot` project.

This file mirrors `CLAUDE.md` (which `@AGENTS.md`s in to it). Any meaningful
change here must be reflected in both projects' Claude/Agents docs if the
behaviour spans both repos.

---

## What this project is

A read-only website that surfaces all-time data for **The League** (ESPN
league id `1917791320`), targeting an audience of ~12 managers who view
exclusively from their phones. Designed to **install as a PWA** ("Add to
Home Screen" → fullscreen, app-icon launcher, dark theme).

Sister project: [`sportsbook-bot`](../sportsbook-bot) — Telegram bot that
shares the same league surface area; both reach ESPN through a similar
`fantasy_espn` module and both implement the **owner identity resolution**
algorithm described below. Keep the two implementations in sync.

---

## Stack

- **Next.js 16 (App Router)** — fully prerendered with `force-static`.
- **Static export** — `next.config.ts` sets `output: "export"`, so production builds emit an `out/` directory.
- **TypeScript** — strict.
- **Tailwind CSS v4** — design tokens declared in `globals.css` via `@theme inline`.
- **lucide-react** — icon set used by the bottom tab bar.
- **Python 3.12 + `espn-api`** — data pipeline only (`scripts/build_data.py`).
- **GitHub Actions** — hourly cron to regenerate JSON + commit it back.
- **Cloudflare Pages** (planned) — hosting target. Free tier, fast CDN.

No database, no API server, no auth — the entire site is a static export
hydrated from JSON files in `/public/data`.

## Architecture

```
ESPN unofficial Fantasy API
         │  (espn-api Python lib)
         ▼
scripts/build_data.py
         │  writes JSON
         ▼
public/data/*.json     ←  also committed to git so the site can be rebuilt
         │              from a clean checkout without ESPN access.
         ▼
src/lib/data.ts (typed loaders)
         │
         ▼
src/app/**/page.tsx   ←  Server Components, force-static.
         │
         ▼
Static HTML + PWA manifest
         │
         ▼
Cloudflare Pages CDN
```

**Data is regenerated by a separate Python process, never by the site at
runtime.** This keeps the site hostable on any static CDN and lets the data
refresh independently of code deploys.

## Owner identity resolution (read this before touching `build_data.py`)

ESPN's per-team owner records are inconsistent across seasons:

- **Same person, different ESPN UUIDs** — managers who delete/recreate their
  ESPN account get a brand-new owner id. Example: Ricky Cano played 2023
  under one UUID and 2025 under a different one.
- **Different people, identical First-Initial** — "Kevin Liang" and
  "Kevin Llerena" are both rendered as "Kevin L." by the naive
  `firstName + last[0]` formula. This actively merged their stats in v1.

`scripts/build_data.py::build_owner_resolution()` solves both:

1. **Auto-merge** — group raw ESPN owner ids by lowercased
   `(firstName, lastName)`. All ids in a group collapse to one canonical id.
2. **Per-owner shortest unique label** — for each canonical owner, find the
   shortest disambiguating last-name suffix that is globally unique. So
   most owners stay "First L." (1 letter); only the two Kevins expand to
   "Kevin L." (Liang) and "Kevin Ll." (Llerena).
3. **Slug collision guard** — if two slugs still collide for some reason,
   the later one gets `-2` appended.

**Trade-off**: two unrelated humans with identical first+last names would
get falsely merged. Acceptable risk in a 12-person league. If it ever
happens, add an explicit override in this file and split them by raw id.

The bot project mirrors this in `fantasy_espn.py::resolve_owner_display`.
**Keep both implementations in sync if you change the algorithm.**

## Mobile-first design rules

When adding new pages:

- Use `force-static` page export so everything pre-renders.
- Pad page top with `pt-6` and bottom with `pb-4` (the layout already
  reserves space for the bottom tab bar via the body's `pb-[calc(64px+...)]`).
- Cards: `rounded-2xl border border-app bg-elev p-3-to-4`.
- Lists are vertical, full-width, big tap targets (≥44px high). Use
  `active:bg-elev-2` for tap feedback (no hover states — touch-only).
- Numbers: `tabular-nums` so digits don't shift around.
- Don't introduce horizontal scrolling. Truncate long names with `truncate`.
- Section labels: `text-[11px] font-bold tracking-widest text-accent`.
- Manager links go to `/managers/[slug]` — never link by team name.
- Avoid emoji-only signal in critical UI; pair with text/colour for accessibility.

## Local development

```bash
cd theleague-web
npm install               # one-time
npm run dev               # http://localhost:3000
npm run build             # static export, sanity check
```

Refreshing data:
```bash
py -3 scripts/build_data.py
```

To preview on a phone on the same WiFi:
```bash
ipconfig                  # find IPv4 Address
# open http://YOUR_IP:3000 on the phone (allow Node through Windows firewall on first run)
```

## Secrets

All ESPN cookies are currently **inlined as defaults** in
`scripts/build_data.py` so the script runs out-of-the-box for local dev. In
production these MUST come from env vars (the GitHub Action passes them as
secrets). Rotate roughly yearly when ESPN cookies expire.

The committed defaults are tied to a single ESPN account (`SWID
{9A38199A-...}`). Don't paste them into other repos.
